# Decoradores

Siempre comienzan con @

### @export:

Hacen que una variable puedas editarla en el inspector:

```python
# En el nodo al que le asignaste el script, podras ver una propiedad de nombre "numero", con un 16 que puedes modificar
@export var numero: int = 16
```

### @onready:

Difiere la inicialización de una variable hasta el evento **`_ready()`**, evitando errores si los nodos hijos no existen aún.
Es **OBLIGATORIO** utilizarlo si vas a llamar a un nodo.

```python
# Si pones $ te apareceran todos los nodos hijos del que tiene el script
@onready var imagen: Sprite2D = $Sprite2D
# Si no lo utilizas en esta ocasion Godot dara un error
```

### @export_range:

Decorador especializado que limita los valores editables de una variable en el Inspector a un rango específico, mostrando un slider para mayor precisión.

```python
@export_range(0, 100) var vida: int  # Enteros 0-100
@export_range(0.0, 10.0, 0.5) var velocidad: float  # Flotantes, incrementos de 0.5
```

### @tool:

Permite que un script se ejecute en el **editor** (modo tool), no solo en runtime. Ideal para generar nodos dinámicamente o mostrar datos en tiempo real mientras editas.

```python
@tool
extends Node2D

# Codigo
```
