# Joint2D

**Joint2D** une **dos nodos físicos** (RigidBody2D, CharacterBody2D) con **restricciones físicas**: bisagras, resortes, cadenas. **Hijo de Node2D**, conecta automáticamente física entre objetos.

## Tipos Principales de Joint2D

| Tipo | Uso | Ejemplo |
| --- | --- | --- |
| **PinJoint2D** | **Punto fijo rotatorio** | Columpio, rueda eje |
| **DampedSpringJoint2D** | **Resorte elástico** | Cuerda elástica, bungee |
| **GrooveJoint2D** | **Carril lineal** | Puerta corredera |

## Configuración Basica

Node2D (Contenedor Joints)
├── StaticBody2D (Ancla/Pared)
│   └── CollisionShape2D
├── RigidBody2D (Caja colgante)
│   └── CollisionShape2D
└── PinJoint2D ← Une los dos arriba
NodeA → StaticBody2D
NodeB → RigidBody2D

## Código Minimo

```python
extends Node2D

@onready var joint = $PinJoint2D
@onready var caja = $RigidBody2D

func _ready():
    # Conectar dinámicamente
    joint.node_a = $StaticBody2D.get_path()
    joint.node_b = caja.get_path()
    
    # Empujar para probar
    caja.apply_impulse(Vector2.RIGHT * 500)
```

## Propiedades Comunes

Node A / Node B → Nodos a unir (OBLIGATORIO)
Bias → Velocidad de corrección (0-1)
Disable Collisions → Evita colisión entre unidos

## Casos de uso Perfectos

✅ Columpios, péndulos
✅ Puertas físicas

✅ Cadenas, sogas
✅ Ruedas con eje
✅ Puentes colgantes
✅ Máquinas rube goldberg

**Ventaja única**: **Física realista automática**. Los objetos se balancean, tensan, oscilan solos.

**Regla**: **SI dos objetos necesitan unión física = Joint2D.**
